# Finspark RAG 平台 — 完整界面方案

> 版本：v2.0 | 日期：2026-03-30  
> 定位：将现有 RAG 知识库升级为**企业级 RAG 平台**，覆盖数据处理、检索优化、评测运维、知识沉淀和版本管理的全生命周期管理。  
> 更新：v2.0 新增对话知识沉淀 (P.14)、知识库健康度检查 (P.15)、知识库版本管理 (P.16) 三个模块。

---

## 一、平台总体架构与导航结构

### 1.1 平台定位

Finspark RAG Platform 不是一个简单的"知识库问答页"，而是一个**端到端的 RAG 工程平台**，面向两类用户：

| 角色 | 核心诉求 | 使用频率 |
|------|---------|---------|
| **业务用户**（投研分析师） | 上传财报 → 提问 → 获取可信答案 | 每日高频 |
| **平台运营/开发者** | 优化检索质量、调试 Prompt、评测效果、监控日志 | 每周中频 |

#### 核心交互要素

| 区域 | 组件 | 说明 |
|------|------|------|
| 上传区域 | 拖拽上传 / 点击选择 / 文本粘贴 Tab 切换 | 支持 `.pdf` 文件，最大 50MB；文本粘贴保留现有功能，最大 500K 字符 |
| 文档信息 | 表单字段 | 标题（必填）、股票代码、公司名称、分类下拉（年报/季报/研报/公告/通用）、标签多选 |
| 解析引擎 | Radio 选择 | MinerU 云端 OCR（推荐，支持扫描件） vs 纯文本提取（快速，仅适合可选文字 PDF）|
| 切片策略 | Radio 选择 | 5 种策略：递归字符分割 / Token级分割 / 语义段落 / 按页面 / Markdown行分割 |
| 切片参数 | 滑动条 + 数字输入 | Chunk Size（100~2000，默认500）、Overlap（0~500，默认100）、最大Chunk数（可选） |
| 高级选项 | Checkbox 组 | 页码保留、表格独立、LLM 表格序列化、去除页眉页脚、合并短段落 |
| Embedding 配置 | 下拉选择（只读展示） | 显示当前系统配置的 Provider/Model/维度，未来可支持选择 |
| 切片预览 | 实时列表 | 上传文件 + 配置参数后，点击"预览切片结果"按钮，异步展示前 N 个 Chunk 的内容、Token数、页码范围 |
| 处理进度 | 多步进度条 | 提交入库后显示 4 步进度：PDF解析 → 文本分块 → 向量化 → 索引构建 |

#### 关键业务规则

1. PDF 文件上传后，先调用 MinerU API 异步解析，前端轮询任务状态
2. 切片参数修改后，"预览切片结果"区域实时刷新（仅展示前 20 个 Chunk）
3. "确认入库" 后进入 4 步处理流水线，每步有独立进度条
4. 如果启用了 "LLM 表格序列化"，则在分块阶段之后增加一步 "表格序列化" 处理
5. 同一文档标题 + 股票代码不可重复上传（需提示覆盖确认）

#### 核心功能说明

| 功能 | 说明 | 对应代码 |
|------|------|---------|
| **创建版本** | 对当前知识库（所有文档+Chunk+向量索引）创建不可变快照。每个版本独立存储元数据和统计信息 | `create_version()` |
| **版本时间线** | 按时间排列的版本列表，显示版本号、描述、创建时间、切片数、评测得分、状态（当前/已归档） | `versions` 字典 |
| **版本 Diff 对比** | 选择两个版本，自动检测变更：新增/删除/修改/未变 的切片列表，以及统计指标对比 | `compare_versions()` + `detect_changes()` |
| **统计对比** | 切片总数、总内容长度、平均切片长度等数值指标的逐项对比和百分比变化 | `compare_statistics()` |
| **性能评测** | 使用测试集对指定版本运行 FAISS 向量检索，统计准确率和平均响应时间 | `evaluate_version_performance()` |
| **版本性能对比** | 两个版本在同一测试集上的性能指标对比，含逐题结果和自动推荐建议 | `compare_version_performance()` |
| **回归测试** | 运行标准测试集，输出通过/失败列表和通过率，防止知识库变更引入退化 | `generate_regression_test()` |
| **版本回滚** | 将知识库回滚到指定历史版本，恢复该版本的所有切片和向量索引 | 新增 API |
| **性能建议** | 基于两版本的准确率和响应时间自动生成"推荐使用"建议 | `generate_performance_recommendation()` |
